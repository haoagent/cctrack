# Session Log — 2026-03-20 (Session 2, Final)

## Summary
从"理解 AgentPay 项目"出发，完成了一轮完整的产品重构 brainstorming。通过 Agent Team（brainstormer + spec-reviewer）完成了设计 spec v4。

## 关键决策

### 1. 产品名
- **Clipay** (clipay.com)，内部代号 AgentPay

### 2. 定位
- Agent CLI 支付基础设施，面向 Agent + Human Owner
- B2B/B2Dev first，不是 B2C
- CLI 命令就是产品语言，不需要给支付类型起品牌名

### 3. 四层支付分类 → 只做两层
- 创始人提出 4 类（Usage / Purchase / Subscription / Invoice）
- Brainstormer 独立验证后建议：Phase 1 只做 Usage Pay + Purchase Pay
- Subscription Pay → 用定期 Purchase Pay 替代
- Invoice Pay → 独立产品级复杂度，不是 feature
- 只留 `payment_type` 可扩展字段

### 4. 竞品验证
- x402 (Coinbase): 1 种模式，1500万+ 交易
- Nevermined: 1-2 种
- Skyfire: 1 种
- 结论：agent 支付初创都从 1-2 种做起

### 5. 命名
- Usage Pay / Purchase Pay 是内部架构概念
- 对外就是 `clipay subscribe / buy / usage / confirm` 命令
- 不需要给支付类型起对外品牌名

### 6. 设计原则
- 始终以 Agent CLI 视角模拟交互
- 展示 CLI 命令输入输出，不用抽象架构术语
