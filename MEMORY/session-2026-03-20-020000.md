# Session Log: cctrack (formerly AgentTrack) — Design + Implementation + Bugfixes

**Date**: 2026-03-20 02:00

## Summary
从零到可用 MVP：设计、实现、测试、安装、真机调试 cctrack — Claude Code agent team 实时监控工具。

## Key Decisions
1. **产品名**: AgentTrack → cctrack (CC = Claude Code)
2. **技术栈**: Rust + Ratatui + axum（AI coding 抹平 Rust 开发速度劣势）
3. **数据源**: 文件监听 (~/.claude/ JSON) + PostToolUse hooks (stdin JSON via curl -d @-)
4. **Hook 格式**: 发现 Claude Code hook 通过 stdin 传 JSON，不是环境变量
5. **idle_notification 格式**: 发现实际数据中 type 嵌套在 text 字段的 JSON 字符串里，不是顶层字段
6. **Web server 默认启动**: 不需要 --web flag，localhost:7891 始终可用
7. **Hooks 自动安装**: cctrack 首次启动自动装 hooks，零配置

## Bugs Found & Fixed
- idle_notification 检测失败 → 解析 text 字段内的嵌入 JSON
- Messages 面板显示 raw JSON → 过滤 idle_notification 类型
- Hook 命令格式错误（环境变量 → stdin @-）
- Hook 结构错误（flat → nested {matcher, hooks[]}）
- Web server 不默认启动 → 改为始终启动
- PATH 问题 → 添加 ~/.cargo/bin 到 .bash_profile
