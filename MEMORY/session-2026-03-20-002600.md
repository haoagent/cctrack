# Session Log — 2026-03-20 (Session 2)

## Summary
Deep brainstorming session on AgentPay product restructuring. Used Agent Team (brainstormer + spec-reviewer) to research, write, and critique a design spec.

## Key Decisions

### 1. Product Positioning
- AgentPay = **Agent CLI + Human Owner 的支付基础设施**
- B2B/B2Dev first，不是 B2C
- 面向 Agent 和它的 Human Owner，不是面向"支付平台"

### 2. Payment Types — 2 types for Phase 1
- **Usage Pay（用量付费）** — 按调用量计费，API Key 认证
- **Purchase Pay（购买付费）** — 一次性购买，escrow + delivery
- "Usage Pay / Purchase Pay" 是内部架构概念，不是对外品牌
- CLI 命令本身就是最好的命名：`subscribe`, `buy`, `usage`, `confirm`

### 3. 砍掉 Phase 2-3 的预承诺
- Subscription Pay → 可以用定期触发的 Purchase Pay 替代
- Invoice Pay → 是独立产品级别的复杂度，不是 feature
- 只留一个 `payment_type` 可扩展字段，其余不预设计

### 4. 竞品分析
- x402 (Coinbase): 1 种模式，1500万+ 交易
- Nevermined: 1-2 种
- Skyfire: 1 种
- 结论：agent 支付初创都从 1-2 种做起
