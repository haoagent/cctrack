# Session Log: 2026-03-23 16:25

## Summary
Compared cctrack vs ccusage pricing accuracy and fixed two bugs in cctrack's cost calculation.

## Key Decisions
- **Tiered pricing adopted**: Opus 4.6 charges 2x for tokens above 200k per-message (matching LiteLLM/ccusage). Base rates: input $5→$10, output $25→$37.50, cache_write $6.25→$12.50, cache_read $0.50→$1.00
- **Per-message cost calculation**: Cost must be computed per-message (before aggregation) for tiered pricing to be correct. Changed from "aggregate tokens then price" to "price each message then aggregate cost"
- **Live session exclusion**: `cctrack stats` CLI now skips transcripts modified within last 60 seconds to avoid counting in-progress sessions
- **Multi-day session attribution**: cctrack attributes tokens to actual message date (per-day split), ccusage attributes to session start date. cctrack's approach is more granular — not a bug
- **Model tracking per-message**: Changed from "first model wins" to updating model per message, since sessions can switch models mid-conversation

## Files Modified
- `cctrack/src/stats.rs` — Tiered pricing (ModelPricing struct + tiered_cost fn), per-message cost calc in parse_transcript, live session 60s mtime filter, simplified cache_write fields
- `cctrack/src/store/models.rs` — Added `cost_usd` field to TokenUsage, `tiered_cost()` fn, `add_message()` method for per-message tiered pricing
- `cctrack/src/collector/hook_server.rs` — Updated `read_transcript_usage_and_model()` to use `TokenUsage::add_message()` for per-message tiered pricing
- `cctrack/src/web/static/app.js` — `estimateCost()` now uses pre-computed `cost_usd` from server when available

## Pricing Comparison Results
- Before fix: cctrack $172 (flat) vs ccusage $135 (tiered) — cctrack overcounted tokens, undercounted per-token cost
- After fix: cctrack $230 (tiered, excl live) vs ccusage $146 (tiered, incl live) — remaining gap from multi-day session date attribution
