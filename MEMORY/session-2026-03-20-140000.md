## Summary
完成 Clipay Rust Phase 1 MVP 全部实现：从 JS 参考代码出发，用 Rust (Axum + rusqlite + clap) 重写了完整的 Usage Pay + Purchase Pay 双模式支付系统。

## Key Decisions
1. **Workspace 结构**: `clipay-core` (业务逻辑) + `clipay-server` (Axum HTTP) + `clipay-cli` (clap CLI)
2. **DB 设计**: 9 张表 — 原有 7 张 + api_keys + usage_records，加了 payment_type 和 Usage Pay 扩展字段
3. **Bug Fixes 落地**:
   - Provider credit timing: 只在 provider 成功返回后 credit
   - Transaction status: 所有错误路径更新为 'refunded'
   - Daily limits: Usage Pay 中强制检查 usage_daily_limit
4. **API Key 安全**: SHA-256 hash 存储，原始 key 只返回一次
5. **Rate Limiting**: 内存级 per-key per-minute 计数器

## API Endpoints (20+)
### Usage Pay (new)
- POST `/api/subscribe/:provider` — 订阅，返回 API key
- POST `/api/usage-pay/:provider` — 计量调用 (X-AgentPay-Key header)
- GET `/api/keys?wallet=xxx` — 列出 API keys
- DELETE `/api/keys/:keyId` — 吊销 key
- GET `/api/usage?wallet=xxx` — 用量统计

### Purchase Pay (renamed from MVP)
- POST `/api/buy/:provider` — 购买 (escrow + delivery)
- POST `/api/deliver/:jobId` — 异步交付
- GET `/api/jobs/:jobId` — 查询 job 状态
