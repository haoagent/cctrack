# Session Log: 2026-03-23 16:38

## Summary
Sub-agent 可见性调试 + 修复。发现根因是 sub-agent hook payload 的 session_id 和 parent 相同，cctrack 无法区分。改用 transcript_path 路径结构检测，同时优化了 tab 显示和超时逻辑。

## Key Decisions
- **Sub-agent 检测方案**: 从 transcript_path 路径解析 (`/subagents/agent-{id}.jsonl`)，替代原来脆弱的 `pending_spawns` CWD 匹配
- **Hook payload 优先**: 如果 hook 直接带 `agent_id`/`agent_type` 字段则优先使用，transcript_path 解析作为 fallback
- **subagent_info 3-tuple**: `(parent_session_id, agent_id, Option<agent_type>)`
- **ALL tab 不显示 sub-agent**: 用 `child_sessions` HashSet 过滤
- **Tab 名截断**: 去掉 `SESSION:` 前缀，max 24 显示字符，CJK 双宽度感知
- **Session tab 无 STATUS 列**: 状态 dot 内嵌到 NAME，列为 `● NAME | MODEL | TOKENS | COST`
- **超时统一**: Idle 2min / Shutdown 10min（不区分 parent/sub-agent）
- **Tab 消失**: Session tab 30min, Team tab 24h
- **Cache token 拆分**: `cache_create_tokens` → `cache_create_5m_tokens` + `cache_create_1h_tokens`（匹配 Anthropic 分级计价）
